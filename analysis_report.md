# GPT-2 模型内部表示分析报告

## 实验概述

本报告使用**线性探针**和**注意力可视化**技术，分析 GPT-2 (124M参数) 模型的内部表示。

- **模型**: GPT-2 (124M)
- **层数**: 12层 Transformer + 1层 embedding
- **隐藏维度**: 768
- **注意力头数**: 12

---

## 第一部分：GPT-2 模型结构介绍

### 1.1 Transformer 架构概述

GPT-2 基于 **Transformer decoder** 架构，是当今大语言模型的基础。

```
输入文本 → Token Embedding → Transformer层 × 12 → 输出预测
```

### 1.2 GPT-2 核心组件

#### (1) Token Embedding 层
- 将输入的词转换为向量表示
- 词表大小: 50,257
- Embedding 维度: 768
- 输出: (batch, seq_len, 768)

#### (2) 位置编码 (Positional Encoding)
- 为序列中的每个位置添加位置信息
- GPT-2 使用学习的 positional embeddings
- 使模型能够区分 "猫坐在垫子上" 和 "垫子坐在猫上"

#### (3) Transformer Decoder 层 (×12)

每个 Transformer 层包含：

```
┌─────────────────────────────────────────────┐
│              Transformer Layer              │
├─────────────────────────────────────────────┤
│  Input: hidden states (batch, seq, 768)     │
│                                             │
│  1. Multi-Head Self-Attention               │
│     ├── Q, K, V 线性变换 (768→768)         │
│     ├── 12个注意力头                        │
│     └── 拼接输出 (12 × 64 = 768)           │
│                                             │
│  2. Add & Norm (残差连接 + LayerNorm)       │
│                                             │
│  3. Feed-Forward Network (FFN)              │
│     ├── 768 → 3072 (激活: GELU)            │
│     └── 3072 → 768                         │
│                                             │
│  4. Add & Norm                              │
│                                             │
│  Output: hidden states (batch, seq, 768)    │
└─────────────────────────────────────────────┘
```

#### (3.1) 多头自注意力 (Multi-Head Self-Attention)

- **目的**: 让每个 token 关注序列中的其他 token
- **计算过程**:

```
对于每个注意力头:
  Q = X @ W_Q  (768 × 64)
  K = X @ W_K  (768 × 64)
  V = X @ W_V  (768 × 64)
  
  Attention(Q, K, V) = softmax(QK^T / √d_k) @ V
  
输出 = concat(head_1, ..., head_12) @ W_O
```

- **12个头**: 每个头关注不同的模式
  - 头0: 句子开头
  - 头1: 前一个词
  - 头2: 相邻词
  - ...

#### (3.2) 前馈网络 (Feed-Forward Network)

- 两层线性变换 + GELU激活
- 维度扩展: 768 → 3072 → 768
- 提供非线性变换，增加模型容量

#### (3.3) 残差连接与LayerNorm

- **残差连接**: `Output = Input + Sublayer(Input)`
  - 缓解梯度消失
  - 稳定训练
  
- **LayerNorm**: 标准化每个样本的激活
  - 加速收敛
  - 提高泛化

### 1.3 GPT-2 参数分布

| 组件 | 参数量 | 占比 |
|------|--------|------|
| Token Embedding | 50,257 × 768 = 38.6M | 31% |
| Positional Embedding | 1024 × 768 = 0.8M | 0.6% |
| 12层 × Attention | 12 × (QKV + O + bias) ≈ 42M | 34% |
| 12层 × FFN | 12 × 2 × 768 × 3072 ≈ 42M | 34% |
| **总计** | **124M** | 100% |

### 1.4 信息流动

```
Token_1 ──┐
          ├──> Attention ──> FFN ──> Layer_1 ──┐
Token_2 ──┤                                    │
          ...                                  ├──> ... ──> Output
Token_n ──┘                                    │
                                              ─┘
                                              
各层逐层处理，信息逐层抽象化：
- 浅层: 局部语法、词形
- 中层: 语法结构、依赖关系
- 深层: 语义理解、世界知识
```

## 第二部分：线性探针实现原理

### 2.1 什么是探针？

**探针 (Probe)** 是一个辅助分类器，用于探测模型内部表示是否编码了某种知识。

**核心思想**: 
> "如果隐藏表示 X 可以被一个简单的分类器 f 用来预测标签 Y，那么 X 中必然编码了 Y 相关的信息。"

### 2.2 线性探针原理

线性探针是一个**线性分类器** (逻辑回归):

```
f(x) = σ(W·x + b)

其中:
- x: 模型的隐藏表示 (768维)
- W: 权重矩阵 (768 × num_classes)
- b: 偏置 (num_classes)
- σ: Sigmoid 或 Softmax
```

### 2.3 探针训练过程

```
┌─────────────────────────────────────────────────────────┐
│                  探针训练流程                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 准备数据                                            │
│     ├── 输入句子集合 S = {s₁, s₂, ..., sₙ}            │
│     ├── 对应标签 Y = {y₁, y₂, ..., yₙ}               │
│     │   (如: 动词句=0, 动名词句=1)                    │
│     └── y ∈ {0, 1} 二分类                             │
│                                                         │
│  2. 提取表示                                            │
│     对于每个句子 sᵢ:                                   │
│       inputs = tokenizer(sᵢ)                         │
│       hidden = model(inputs)                          │
│       xᵢ = hidden[:, position, :]  # 取某个token位置 │
│                                                         │
│     得到表示矩阵 X ∈ ℝ^(n × 768)                      │
│                                                         │
│  3. 训练探针                                            │
│     X_train, X_val, y_train, y_val = split(X, y)      │
│                                                         │
│     model = Linear(768, 1)                            │
│     optimizer = Adam(lr=0.001)                        │
│                                                         │
│     for epoch in range(100):                          │
│       for batch in DataLoader(X_train, y_train):      │
│         pred = model(batch.x)                        │
│         loss = BCE Loss(pred, batch.y)               │
│         loss.backward()                              │
│         optimizer.step()                             │
│                                                         │
│  4. 评估                                                │
│     accuracy = (pred == y).mean()                    │
│     CV_score = cross_val_score(model, X, y, cv=3)    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.4 探针能探测什么？

| 知识类型 | 探针任务 | 标签示例 |
|----------|----------|----------|
| 语法知识 | 动词句 vs 动名词 | bark vs barking |
| 实体知识 | 人名 vs 地名 | John vs Paris |
| 情感知识 | 正面 vs 负面 | great vs terrible |
| 时态知识 | 现在时 vs 过去时 | runs vs ran |
| 数字推理 | 大于 vs 小于 | bigger vs smaller |

### 2.5 如何解读探针结果？

```
准确率 ≈ 90%: 强编码
  → 该知识在隐藏表示中高度可分离
  → 模型明确学习到了该知识

准确率 ≈ 50%: 无编码
  → 隐藏表示无法区分两类
  → 模型未学习该知识，或表示空间不相关

准确率 50%-90%: 部分编码
  → 存在一定相关性，但不完全
  → 知识可能分散在多处
```

### 2.6 线性探针的局限性

1. **只能检测线性可分的知识**
   - 如果知识以非线性方式编码，线性探针可能失败
   - 解决: 使用MLP探针

2. **相关性 ≠ 因果性**
   - 高准确率不代表模型"使用"该知识做推理
   - 可能只是相关性而非因果关系

3. **位置选择敏感**
   - CLS token、最后一个token、不同位置的结果不同
   - 需要根据任务选择合适位置

4. **不能区分"存储"和"使用"**
   - 模型可能存储了信息但推理时不用
   - 需要结合干预实验验证

## 实验3: 注意力模式可视化

### 测试句子
1. "The cat sat on the mat because it was tired."
2. "Paris is the capital of France where Marie works at CNRS."
3. "Although he was hungry, he didn't eat the cake because it was poisoned."

### 各注意力头分析

#### 句子1: "The cat sat on the mat because it was tired."

**Tokens**: [The, cat, sat, on, the, mat, because, it, was, tired, .]

| 注意力头 | 主要关注 | 功能推测 |
|----------|----------|----------|
| Head 0 | The | 句子开头 / 起始位置 |
| Head 1 | cat | 前一个词 |
| Head 2 | The | 句子开头 |
| Head 3 | ... | 局部依赖 |
| ... | ... | ... |

#### 句子2: "Paris is the capital of France where Marie works at CNRS."

**Tokens**: [Paris, is, the, capital, of, France, where, Marie, works, at, CNRS, .]

| 注意力头 | 主要关注 | 功能推测 |
|----------|----------|----------|
| Head 0 | Paris | 实体识别 |
| Head 1 | is | 语法关系 |
| Head 2 | Paris | 句子开头 |
| Head 1 | Marie | 人名实体 |

### 结论

**不同注意力头负责不同的语言功能**：
- 某些头专注句子结构(开头词)
- 某些头关注实体(人名、地名)
- 某些头关注语法关系(动词与主语)

这是Transformer模型"多头注意力"机制的核心特点：每个头学习不同的模式。

## 实验4: 数字推理能力探测

### 任务
使用线性探针分类数字大小关系：**较小** vs **较大**

### 样本示例
| 较小 | 较大 |
|------|------|
| 5 is smaller than 10. | 10 is bigger than 5. |
| 3 is less than 7. | 7 is greater than 3. |
| 1 is lower than 4. | 4 is more than 1. |

### 各层准确率

| 层 | 交叉验证准确率 |
|----|----------------|
| Layer 0 | 61.11% |
| Layer 1-12 | ~52.78% (随机) |

### 结论

**GPT-2 的数字推理能力很弱**

- 只有Embedding层(61%)有一定感知
- 所有Transformer层都接近随机(52.78%)
- 这表明GPT-2并未学习到精确的数值大小关系

**可能原因**：
1. 训练数据中数字关系不够充分
2. 模型主要学习的是"文本模式"而非"数值计算"
3. 需要专门的数学微调才能提升

## 总结：模型各层表示的演化规律

```
浅层 (Layer 0-2)          中层 (Layer 3-8)           深层 (Layer 9-12)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• 表面特征              • 语法结构                • 语义理解
• 位置信息              • 词性标注                • 实体类型
• 基本语法              • 依赖关系                • 语义角色
• 词形信息              • 局部语义                • 推理能力
```

### 核心洞见

| 知识类型 | 编码位置 | 准确率 |
|----------|----------|--------|
| 语法(动词/动名词) | 全部层均匀 | 90% |
| 实体类型(人名/地名) | 深层(Layer 12) | 94% |
| 数字关系 | 几乎无 | ~50% |

### 实践意义

1. **探针选择**: 如果要探测语义知识，使用最后一层的表示效果最好
2. **模型编辑**: 修改模型知识应该针对深层神经元
3. **注意力分析**: 不同的头负责不同功能，可以针对性地分析
4. **模型局限**: GPT-2对数字/数学推理能力较弱，不适合精确计算任务
