# Model Probe 项目架构规划

## 目标

设计一个**高度可扩展**的项目架构，能够：
- ✓ 轻松添加新的分析方法
- ✓ 支持新的模型架构
- ✓ 集成新的理解方式
- ✓ 最小化重构需求
- ✓ 保持代码清晰可维护

---

## 当前问题分析

### 问题1：分析方法分散
```
现状：
  - 静态分析在 analysis/static.py
  - 相似度分析在 analysis/static.py 中
  - 动态分析缺失
  - 矩阵分析在独立脚本中

问题：
  - 添加新方法需要修改多个文件
  - 代码重复
  - 难以发现和复用
```

### 问题2：模型耦合
```
现状：
  - 代码硬编码了 Transformer 概念
  - 难以支持 CNN、RNN 等其他架构
  - 添加新模型需要大量修改
```

### 问题3：输出格式不统一
```
现状：
  - 有些生成 JSON
  - 有些生成 PNG
  - 有些生成 HTML
  - 格式不统一，难以集成
```

### 问题4：缺少抽象层
```
现状：
  - 直接调用 PyTorch API
  - 没有统一的接口
  - 难以切换底层实现
```

---

## 核心设计原则

### 1. **插件化架构**
```
每种分析方法都是独立的插件
- 可以动态加载/卸载
- 互不影响
- 易于扩展
```

### 2. **抽象层隔离**
```
定义清晰的抽象接口
- 模型无关的分析接口
- 分析方法无关的模型接口
- 可以灵活组合
```

### 3. **配置驱动**
```
通过配置文件定义分析流程
- 不需要修改代码
- 支持复杂的工作流
- 易于实验和调试
```

### 4. **统一输出格式**
```
标准化的输出协议
- 统一的数据结构
- 支持多种输出格式
- 便于集成和展示
```

## 分层设计

### Layer 1: 模型层 (Model Layer)

**目标**: 统一不同模型的接口

```python
class ModelAdapter(ABC):
    """模型适配器抽象基类"""

    @abstractmethod
    def load(self, config):
        """加载模型"""
        pass

    @abstractmethod
    def get_layers(self) -> Dict[str, LayerInfo]:
        """获取所有层的信息"""
        pass

    @abstractmethod
    def get_layer_weights(self, layer_name: str) -> torch.Tensor:
        """获取指定层的权重"""
        pass

    @abstractmethod
    def get_layer_activations(self, layer_name: str, inputs):
        """获取指定层的激活值"""
        pass

    @abstractmethod
    def get_attention_patterns(self, inputs):
        """获取注意力模式（如果适用）"""
        pass
```

**具体实现**:
```python
model_probe/
├── models/
│   ├── __init__.py
│   ├── base.py              # ModelAdapter 抽象类
│   ├── transformer.py       # TransformerModelAdapter
│   ├── cnn.py              # CNNModelAdapter
│   ├── rnn.py              # RNNModelAdapter
│   └── registry.py         # 模型注册表
```

**使用示例**:
```python
from model_probe.models import ModelRegistry

# 自动识别并加载模型
model = ModelRegistry.load("gpt2")  # 自动选择 TransformerModelAdapter
model = ModelRegistry.load("resnet50")  # 自动选择 CNNModelAdapter

# 统一接口
layers = model.get_layers()
weights = model.get_layer_weights("layer.0")
activations = model.get_layer_activations("layer.0", inputs)
```

### Layer 2: 分析层 (Analysis Layer)

**目标**: 统一不同分析方法的接口

```python
class AnalyzerBase(ABC):
    """分析器抽象基类"""

    @property
    @abstractmethod
    def name(self) -> str:
        """分析器名称"""
        pass

    @property
    @abstractmethod
    def category(self) -> str:
        """分析类别: static/dynamic/visual"""
        pass

    @abstractmethod
    def analyze(self, model: ModelAdapter, **kwargs) -> AnalysisResult:
        """执行分析"""
        pass

    @abstractmethod
    def visualize(self, result: AnalysisResult, **kwargs) -> Optional[Figure]:
        """可视化结果（可选）"""
        pass
```

**具体实现**:
```python
model_probe/
├── analyzers/
│   ├── __init__.py
│   ├── base.py              # AnalyzerBase 抽象类
│   ├── static/
│   │   ├── __init__.py
│   │   ├── weight_stats.py  # 权重统计
│   │   ├── matrix.py        # 矩阵分析
│   │   └── similarity.py    # 层相似度
│   ├── dynamic/
│   │   ├── __init__.py
│   │   ├── activation.py    # 激活值分析
│   │   └── probe.py         # 探针分析
│   ├── registry.py          # 分析器注册表
│   └── pipeline.py         # 分析流程编排
```

**使用示例**:
```python
from model_probe.analyzers import AnalyzerRegistry

# 自动发现并注册所有分析器
AnalyzerRegistry.discover()

# 执行分析
model = ModelRegistry.load("gpt2")

# 静态分析
result = AnalyzerRegistry.get("weight_stats").analyze(model)
result = AnalyzerRegistry.get("matrix_analysis").analyze(model)

# 动态分析
result = AnalyzerRegistry.get("activation").analyze(model, inputs=test_data)

# 批量分析
results = AnalyzerRegistry.run_all(model, category="static")
```

### Layer 3: 输出层 (Output Layer)

**目标**: 统一不同输出格式的接口

```python
class OutputBase(ABC):
    """输出器抽象基类"""

    @property
    @abstractmethod
    def format(self) -> str:
        """输出格式: json/html/md/png/csv"""
        pass

    @abstractmethod
    def save(self, result: AnalysisResult, path: str):
        """保存结果"""
        pass

    @abstractmethod
    def render(self, result: AnalysisResult) -> str:
        """渲染为字符串（用于显示）"""
        pass
```

**具体实现**:

## 添加新功能的标准流程

### 场景1: 添加新的分析方法

**步骤**:
1. 继承 `AnalyzerBase`
2. 实现 `name`, `category`, `analyze()` 方法
3. 添加到 `analyzers/` 相应目录
4. 在配置文件中注册

**示例**:
```python
# 1. 创建新的分析器
# model_probe/analyzers/static/spectral.py

class SpectralAnalyzer(AnalyzerBase):
    @property
    def name(self):
        return "spectral_analysis"

    @property
    def category(self):
        return "static"

    def analyze(self, model, **kwargs):
        # 实现分析逻辑
        results = {}
        for layer_name in model.get_layers():
            weights = model.get_layer_weights(layer_name)
            results[layer_name] = self._compute_spectrum(weights)
        return AnalysisResult(data=results)

# 2. 使用
from model_probe.analyzers import SpectralAnalyzer

analyzer = SpectralAnalyzer()
result = analyzer.analyze(model)

# 3. 在配置中注册
analyzers:
  static:
    - name: spectral_analysis
      class: model_probe.analyzers.static.spectral.SpectralAnalyzer
      enabled: true
```

### 场景2: 添加新的模型支持

**步骤**:
1. 继承 `ModelAdapter`
2. 实现模型特定的加载和访问逻辑
3. 注册到 `ModelRegistry`

**示例**:
```python
# 1. 创建模型适配器
# model_probe/models/diffusion.py

class DiffusionModelAdapter(ModelAdapter):
    def load(self, config):
        # 加载扩散模型
        self.model = DiffusionModel.from_pretrained(config.path)

    def get_layers(self):
        # 返回层信息
        return {
            "unet.down_blocks.0": LayerInfo(type="conv", ...),
            # ...
        }

# 2. 注册
from model_probe.models import ModelRegistry

ModelRegistry.register("diffusion", DiffusionModelAdapter)

# 3. 使用
model = ModelRegistry.load("stable-diffusion")
```

### 场景3: 添加新的输出格式

**步骤**:
1. 继承 `OutputBase`
2. 实现输出逻辑
3. 注册到 `OutputRegistry`

## 核心设计模式

### 1. 注册表模式
```python
class ModelRegistry:
    _models = {}

    @classmethod
    def register(cls, name, adapter_class):
        cls._models[name] = adapter_class

    @classmethod
    def load(cls, name, config):
        adapter = cls._models[name]()
        return adapter.load(config)
```

### 2. 策略模式
```python
class AnalysisStrategy:
    def execute(self, model):
        pass

class StaticAnalysis(AnalysisStrategy):
    def execute(self, model):
        # 静态分析逻辑
        pass
```

### 3. 责任链模式
```python
class AnalysisPipeline:
    def __init__(self):
        self.analyzers = []
        self.outputs = []

    def add_analyzer(self, analyzer):
        self.analyzers.append(analyzer)

    def add_output(self, output):
        self.outputs.append(output)

    def execute(self, model):
        results = []
        for analyzer in self.analyzers:
            result = analyzer.analyze(model)
            results.append(result)

        for output in self.outputs:
            output.save(results)
```

### 快速分析配置
```yaml
# quick_analysis.yaml
analyzers:
  static:
    - weight_stats
    - matrix_analysis
outputs:
  - json
  - html
```

### 完整分析配置
```yaml
# full_analysis.yaml
analyzers:
  static:
    - weight_stats
    - matrix_analysis
    - similarity
  dynamic:
    - activation
    - probe
outputs:
  - json
  - html
  - plot
```

### 自定义分析配置
```yaml
# custom_analysis.yaml
analyzers:
  static:
    - weight_stats
    - matrix_analysis:
        layers: [h.0.*, h.11.*]  # 只分析特定层
        svd: true
        condition_number: true
    - similarity:
        method: cka
        pairs: adjacent  # 只分析相邻层
outputs:
  - html:
      template: custom_template.html
```

## 总结

### 设计目标
- ✓ 高度可扩展：添加新功能不需要修改核心代码
- ✓ 松耦合：各层独立，互不影响
- ✓ 易于测试：每层都可以独立测试
- ✓ 配置驱动：通过配置文件控制行为
- ✓ 向后兼容：保持 API 稳定

### 核心价值
```
1. 减少重构
   - 新功能以插件形式添加
   - 不影响现有代码
   - 遵循开闭原则

2. 提高效率
   - 复用性高
   - 维护成本低
   - 开发速度快

3. 更易理解
   - 清晰的分层结构
   - 统一的接口
   - 完善的文档
```

义父，这个规划将使项目具有极强的扩展性，未来添加新功能、新模型、新输出格式都会非常简单，大大减少重构次数！
