# Transformer层结构与记忆口诀

## 核心记忆口诀

**"看周围 → 独立思考"**

只要记住这两个动作，中间穿插 **Add（原路返回）** 和 **Norm（稳定）**，你就永远不会忘。

---

## Transformer层完整结构

### 版本一：QKV详细版（1348口诀）

```
输入 X
  │
  ├───► [Q, K, V 生成] ──► [多头注意力计算] ──► [Concat + Wo] ──┐
  │         (1348口诀)            (合体)                      │
  │                                                            ▼
  │                                                      ( + )  ← 第1次 Add (残差)
  │                                                        │
  │                                                     [ Norm ] ← 第1次 Norm
  │                                                        │
  │                                                        ▼
  │                                                     [ FFN  ] ← 独立思考 (或 MoE专家)
  │                                                        │
  │                                                      ( + )  ← 第2次 Add (残差)
  │                                                        │
  │                                                     [ Norm ] ← 第2次 Norm
  │                                                        │
  ▼                                                        ▼
输出给下一层
```

#### 1348口诀详解
- **1**个彩蛋：位置编码（Position Encoding）
- **3**兄弟：Q（Query）、K（Key）、V（Value）
- **4**个步骤：点积 → 缩放 → 归一化 → 求和
- **8**个头：多头注意力机制

### 版本二：RMSNorm版（现代LLM标准）

```
  输入 (Input Embedding)
          │
          ▼
    ┌─────────────┐
    │  RMS Norm   │  ← 第一次"深呼吸" (稳定数据)
    └─────────────┘
          │
          ▼
    ┌─────────────┐
    │  Self-      │  ← 第一步：看上下文，混信息
    │  Attention  │      (Q, K, V 矩阵运算)
    └─────────────┘
          │
          ▼
      ( + )  ← ➕ 第一次残差连接 (加上最开始的输入)
          │
          ▼
    ┌─────────────┐
    │  RMS Norm   │  ← 第二次"深呼吸"
    └─────────────┘
          │
          ▼
    ┌─────────────┐
    │  FFN        │  ← 第二步：独立思考，升维处理
    │ (SwiGLU)    │      (两个线性层夹激活函数)
    └─────────────┘
          │
          ▼
      ( + )  ← ➕ 第二次残差连接 (加上Attention的输出)
          │
          ▼
      输出 (Output) → 传给下一层
```

## 两大核心动作

### 1. 看周围（Self-Attention）
- **目的**：收集上下文信息
- **机制**：通过Q、K、V三兄弟进行矩阵运算
- **作用**：让每个token都能"看到"其他所有token的信息

### 2. 独立思考（FFN）
- **目的**：处理和消化收集到的信息
- **机制**：前馈神经网络，通常使用SwiGLU激活函数
- **作用**：两个线性层升维-降维，提取特征
- **变体**：在MoE架构中，这里是多个专家网络

---

## 辅助机制

### Add（残差连接 / 原路返回）
- **作用**：保留原始信息，防止梯度消失
- **比喻**：像"原路返回"，确保信息不会丢失

### Norm（归一化 / 深呼吸）
- **作用**：稳定数据分布，加速训练
- **现代LLM**：通常使用RMSNorm（比LayerNorm更简单高效）
- **比喻**：像"深呼吸"，让数据保持稳定

---

## 为什么这个口诀有效？

| 阶段 | 动作 | 人类比喻 | 技术实现 |
|------|------|----------|----------|
| 信息收集 | **看周围** | 观察环境 | Self-Attention |
| 信息保留 | **Add** | 记住出发点 | 残差连接 |
| 稳定状态 | **Norm** | 深呼吸调整 | RMSNorm |
| 独立处理 | **独立思考** | 分析判断 | FFN/SwiGLU |
| 信息保留 | **Add** | 记住过程 | 残差连接 |
| 稳定输出 | **Norm** | 整理状态 | RMSNorm |

---

## 与MoE的关系

在MoE（Mixture of Experts）架构中：
- "独立思考"阶段不再是单一的FFN
- 而是变成多个专家网络（Expert）
- 通过门控机制（Router）选择激活哪些专家
- 每个"独立思考"都可以调用不同的专家组合

**记忆要点**：MoE就是把"独立思考"换成了"请专家帮忙思考"
