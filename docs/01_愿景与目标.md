# Model Probe 愿景与目标

## 核心使命

**揭开模型黑盒，让 AI 白化与透明化**

从"能用就行"到"真正掌握"，构建可解释、可控制、可改造的神经网络理解框架。

## 目标层次

```
理解 → 掌握 → 利用 → 改造 → 更深理解
  ↑                          │
  └──────────────────────────┘
           持续迭代
```

### 1. 理解
- 知识存储在哪些层/神经元
- 模型如何处理特定信息
- 注意力在关注什么

### 2. 掌握
- 预测模型行为
- 预判失败场景
- 控制信息流动

### 3. 利用
- 优化模型性能
- 指导模型设计
- 辅助特征工程

### 4. 改造
- 精准编辑知识
- 修复错误认知
- 注入新能力

## 理解的目的

| 目的 | 说明 |
|------|------|
| 可控 | 能预测行为，不出意外 |
| 可靠 | 知道何时失败，可靠部署 |
| 可信 | 透明才有信任 |
| 可改 | 出了问题能定位修复 |
| 合规 | 满足法律要求（GDPR等） |

## 技术路线

### 探针体系

**按训练分类：**
- 训练探针：线性探针、MLP探针
- 非训练探针：直接分析激活统计

**按方式分类：**
- 静态分析：直接分析参数/权重（SVD、奇异值分布、FFN结构）
- 动态分析：Hook捕获推理过程的中间激活

**按粒度分类：**
- Token级：单个token表示
- Layer级：整层输出
- Head级：Attention head分析

### 核心模块

```
ModelProbe/
├── 探测 (Probe)       # 找出知识在哪
├── 分析 (Analysis)    # 解释知识是什么
├── 可视化 (Visualize) # 让人类能看懂
├── 编辑 (Editor)      # 可干预、可控修改
└── 验证 (Verify)      # 确认改动效果
```

## 设计原则

1. **模块化** - 各模块可独立使用
2. **通用性** - 支持 Transformer/CNN/RNN 等多种架构
3. **可扩展** - 易于添加新的探针类型和分析方法
4. **实用性** - 能在有限资源下运行（如 16G 显存）
